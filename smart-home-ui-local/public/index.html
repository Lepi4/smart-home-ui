Настройки

Home Assistant подключается автоматически через Supervisor API. URL и long-lived token вводить не нужно.
Адреса Lovelace/карточек теперь задаются отдельно для каждого уровня: Настройки → Уровни / области → Источники Lovelace этого уровня.
PIN не установлен

PIN — 4 цифры. Цифры отображаются при вводе. Сохранённый PIN не показывается.

Масштаб карты меняется двумя пальцами или кнопками +/−. Аппаратный масштаб задаёт базовый размер плана для конкретной панели/экрана. Вид, масштаб, скрытые панели и kiosk mode сохраняются в /data/ui_state.json и восстанавливаются после переустановки add-on, если данные не удалить.

При замене картинок старые image-файлы не backup-ятся автоматически. Если соотношение сторон изменилось, приложение предложит “Бэкап и продолжить” или “Продолжить”. Загружаемые картинки проверяются и оптимизируются с сохранением aspect ratio.

Картинки найденных комнат

Комнаты не создаются вручную. Список берётся из текущих найденных комнат/конфигурации. Для каждой комнаты можно загрузить, заменить или сбросить пользовательскую картинку.

Layout / Обслуживание

Очистить маркеры

Удаляет размещение устройств и датчиков на общем плане и в комнатах. Список устройств, комнаты, зоны, картинки, PIN, dangerous и Attention не затрагиваются.

Расширенные действия с зонами и маркерами
Очистить зоны

Опасное действие: удаляет зоны найденных комнат. Обычно зоны появляются/пересоздаются после сканирования источников или редактируются по комнате.

Пересоздать / назначить зону вручную

Выберите найденную комнату и откройте SVG Zone Editor. Это расширенное admin-действие, а не создание комнаты.

Разместить маркеры

Вторичный инструмент после очистки маркеров. Основной workflow: источники → комнаты → зоны/картинки → датчики → маркеры.

Доступно только в admin mode. Перед очисткой создаётся backup layout.json.

Полный сброс к дефолту

Опасное действие: удаляет пользовательские данные проекта и возвращает приложение к чистому состоянию. Перед выполнением создаётся backup; пользовательский PIN удаляется, встроенный резервный механизм восстановления остаётся.

Комнаты / зоны / стандартные датчики

Комнаты не создаются вручную: список берётся из найденных комнат Lovelace / HA Areas / entity names. Для каждой найденной комнаты можно создать или редактировать зону и задать стандартные датчики. Если все entity датчиков очищены, строка стандартных датчиков комнаты не отображается.

Профили

Профили позволяют иметь до 5 параллельных вариантов проекта: свои картинки, layout, zones, rooms settings, устройства и источники. Security/PIN и Attention остаются общими.

Загрузка профилей...

Уровни / области

Уровни позволяют разделить объект на этажи, мансарду, двор, придомовую территорию и другие области. Название уровня вводится вручную; для каждого уровня отдельно настраиваются карта, зоны, маркеры, картинки и источники Lovelace.

Загрузка уровней...

Backup / архивы

Здесь можно создать ручной backup текущего проекта и удалить накопившиеся backup-и. Удаляются только файлы в /data/backups, активные данные не трогаются.

Источники устройств из панели

Структура и названия берутся из RAW Lovelace / device.txt, состояние — из Home Assistant API. Адрес панели можно указывать как dashboard-unknown или dashboard-unknown/0.
