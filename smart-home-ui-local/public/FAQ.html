ALLHA-2D FAQ — v5.1.0-beta.2

Что такое ALLHA-2D?

ALLHA-2D — это 2D-интерфейс для Home Assistant: общий план, комнаты, устройства, стандартные датчики, виртуальные комнаты, kiosk/mobile режимы и индивидуальные настройки клиентов.

Что изменилось в v5.1.0-beta.2?

  • Добавлена загрузка backup-файла на сервер и последующее восстановление из списка backup.
  • Исправлены потенциальные падения UI при отсутствующих DOM-элементах.
  • Исправлена модальная система Attention/FAQ/Диагностики.
  • Обновлён Service Worker cache до версии v5.1.0-beta.2.
  • Обновлены README, FAQ, prompt, roadmap и скриншоты.
  • Добавлен backup path hardening и безопасное удаление project_documents по LIKE.

Почему на LAN-адресе открывается регистрация клиента?

Адрес http://IP:8099/ предназначен для создания или выбора web-клиента. После регистрации нужно использовать индивидуальную ссылку /client/<slug>, чтобы сохранялись настройки именно этого устройства.

Чем отличается HA Ingress?

В Home Assistant add-on основной UI открывается через Ingress и использует служебный клиент Server. Он не является обычным web-клиентом.

Где мобильный вход?

Мобильное приложение использует порт 32457. Внешний доступ сейчас рассчитан на HTTP через KeenDNS/DDNS/OpenWrt, без обязательного reverse proxy.

Как работает Backup?

В разделе Backup можно создать, скачать, загрузить и восстановить backup. Загрузка принимает .tar.gz/.tgz, скачанный из ALLHA-2D. Restore доступен только admin и требует явного подтверждения.

Почему нельзя использовать baseline/layout в UI?

В пользовательском UI используются формулировки “настройки по умолчанию”, “нормальное состояние”, “расположение датчиков и маркеров” и “расположение плиток”.

Что делать при странном UI после обновления?

Выполните rebuild Docker, затем обновите страницу через Ctrl+F5 или очистите кэш PWA/Service Worker.
