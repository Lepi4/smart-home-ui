FAQ — Smart Home UI Local v3.4.7

Smart Home UI Local работает как Home Assistant add-on через Ingress. URL и long-lived token вводить не нужно: add-on использует Supervisor API.

Что исправлено в v3.4.7

  • Горизонтальная мобильная ориентация теперь определяется не только по ширине, но и по высоте/тач-устройству.
  • Нижняя мобильная панель больше не пропадает в landscape.
  • Верхняя панель стала адаптивной: шрифты и кнопки уменьшаются под размер экрана.
  • Исправлен бесконечный скролл страницы при drag/pinch карты.
  • Для бокового меню Home Assistant задан максимально совместимый panel_icon: mdi:view-dashboard.

Где хранятся данные?

Все runtime-данные хранятся в /data, а не внутри контейнера. Если при удалении add-on в Home Assistant отказаться от удаления данных, после повторной установки layout, импортированные устройства, backups и состояние интерфейса должны восстановиться.

  • /data/layout.json — координаты зон, датчиков и маркеров.
  • /data/backups/ — резервные копии layout.
  • /data/addon_config.json — настройки add-on.
  • /data/source_config.json — настройки источников Lovelace.
  • /data/ui_state.json — последний вид, масштаб, pan, kiosk mode, скрытые панели.
  • /data/devices.js, /data/devices.json, /data/lovelace-source.js, /data/lovelace_raw.json — импортированные устройства и исходник.

Почему маркеры совпадают на ПК и телефоне?

Layout хранит координаты только в процентах 0–100 относительно изображения. Масштаб, pan и hardware scale не записываются в layout.json. Поэтому точка, расставленная на ПК, должна оставаться на той же точке плана на телефоне или планшете.

Как войти в режим редактирования?

Кнопку Редактировать нужно удерживать 2 секунды. В редакторе есть Сохранить изменения и Отменить изменения. Drag не сохраняет layout автоматически.

Как работает управление на touch?

  • Короткий тап — основное действие.
  • Долгое удержание — меню функций.
  • Drag работает только в режиме редактирования.
  • ПКМ на компьютере остаётся только дополнительным shortcut.

Как работает масштабирование?

Двумя пальцами можно увеличивать/уменьшать и двигать карту. Кнопки − / + / ⛶ делают то же самое. Аппаратный масштаб 30–150% задаётся в настройках и применяется как базовый размер карты для конкретного экрана.

Что такое режим киоска?

Киоск скрывает панели и оставляет план/комнату. В киоске есть кнопка Комнаты с пунктом Общий план, чтобы выйти из комнаты, и кнопка выхода из киоска.

Где диагностика?

Настройки → Информация / диагностика. Там доступны вкладки: Система, Entity, Layout, Backups, Команды.

Что проверяет вкладка Layout?

  • pixel-like координаты;
  • координаты вне диапазона 0–100;
  • некорректные точки;
  • количество маркеров, датчиков, зон;
  • кнопка нормализации координат.

Какая Lovelace-панель нужна для сканирования?

Рекомендуется отдельная панель Smart Home UI Source. Каждая карточка entities — это комната; card.title — название комнаты; entity и name — устройство и отображаемое имя.

title: Smart Home UI Source
views:
  - title: Комнаты
    path: smart-home-ui-source
    cards:
      - type: entities
        title: Кухня
        entities:
          - entity: light.kitchen_ceiling
            name: Люстра
          - entity: switch.kitchen_led
            name: Подсветка
          - entity: sensor.kitchen_temperature
            name: Температура

      - type: entities
        title: Гостиная
        entities:
          - entity: light.living_room_main
            name: Основной свет
          - entity: climate.living_room_ac
            name: Кондиционер
          - entity: cover.living_room_curtain
            name: Шторы

Почему иконка в боковом меню может не появиться сразу?

Home Assistant берёт иконку бокового пункта из panel_icon в config.yaml. Иногда sidebar entry кешируется. После обновления add-on стоит обновить репозиторий add-on, перезапустить add-on, удалить пункт из sidebar и добавить его снова. Если не помогает — перезапустить frontend/Home Assistant.
