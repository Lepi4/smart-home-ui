FAQ — ALLHA-2D Local v3.5.8.2

ALLHA-2D Local работает как Home Assistant add-on через Ingress. URL и long-lived token вводить не нужно: add-on использует Supervisor API.

Ребрендинг: название изменено с ALLHA-3D на ALLHA-2D, потому что приложение работает с 2D-планами, зонами и маркерами. Репозиторий и slug сохранены прежними для совместимости.

Что изменилось в v3.5.8.2?

Переключение между уровнями теперь выполняется без полной перезагрузки страницы: должна меняться только карта/слои активного уровня, без промежуточного появления бокового меню ALLHA-2D Local.

На главном экране и в kiosk mode кнопками показаны только другие уровни. Текущий уровень отображается как подпись.

Адреса Lovelace теперь задаются отдельно для каждого уровня: Настройки → Уровни / области → Источники Lovelace этого уровня.

Что исправлено в v3.5.0

  • Горизонтальная мобильная ориентация теперь определяется не только по ширине, но и по высоте/тач-устройству.
  • Нижняя мобильная панель больше не пропадает в landscape.
  • Верхняя панель стала адаптивной: шрифты и кнопки уменьшаются под размер экрана.
  • Исправлен бесконечный скролл страницы при масштабировании/pinch карты.
  • Для бокового меню Home Assistant задан максимально совместимый panel_icon: mdi:view-dashboard.

Где хранятся данные?

Все runtime-данные хранятся в /data, а не внутри контейнера. Если при удалении add-on в Home Assistant отказаться от удаления данных, после повторной установки layout, импортированные устройства, backups и состояние интерфейса должны восстановиться.

  • /data/layout.json — координаты зон, датчиков и маркеров.
  • /data/backups/ — резервные копии layout.
  • /data/addon_config.json — настройки add-on.
  • /data/source_config.json — настройки источников Lovelace.
  • /data/ui_state.json — последний вид, масштаб, pan, kiosk mode, скрытые панели.
  • /data/devices.js, /data/devices.json, /data/lovelace-source.js, /data/lovelace_raw.json — импортированные устройства и исходник.

Почему маркеры совпадают на ПК и телефоне?

Layout хранит координаты только в процентах 0–100 относительно изображения. Масштаб, pan и hardware scale не записываются в layout.json. Поэтому точка, расставленная на ПК, должна оставаться на той же точке плана на телефоне или планшете.

Как войти в режим редактирования?

Кнопку Редактировать нужно удерживать 2 секунды. В редакторе есть Сохранить изменения и Отменить изменения. Drag не сохраняет layout автоматически.

Как работает управление на touch?

  • Короткий тап — основное действие.
  • Долгое удержание — меню функций.
  • Drag работает только в режиме редактирования.
  • ПКМ на компьютере остаётся только дополнительным shortcut.

Как работает масштабирование?

Двумя пальцами можно увеличивать/уменьшать и двигать карту. Кнопки − / + / ⛶ делают то же самое. Аппаратный масштаб 30–150% задаётся в настройках и применяется как базовый размер карты для конкретного экрана.

Что такое режим киоска?

Киоск скрывает панели и оставляет план/комнату. В киоске есть кнопка Комнаты с пунктом Общий план, чтобы выйти из комнаты, и кнопка выхода из киоска.

Где диагностика?

Настройки → Информация / диагностика. Там доступны вкладки: Система, Entity, Layout, Backups, Команды.

Что проверяет вкладка Layout?

  • pixel-like координаты;
  • координаты вне диапазона 0–100;
  • некорректные точки;
  • количество маркеров, датчиков, зон;
  • кнопка нормализации координат.

Какая Lovelace-панель нужна для сканирования?

Рекомендуется отдельная панель ALLHA-2D Source. Каждая карточка entities — это комната; card.title — название комнаты; entity и name — устройство и отображаемое имя.

title: ALLHA-2D Source
views:
  - title: Комнаты
    path: smart-home-ui-source
    cards:
      - type: entities
        title: Кухня
        entities:
          - entity: light.kitchen_ceiling
            name: Люстра
          - entity: switch.kitchen_led
            name: Подсветка
          - entity: sensor.kitchen_temperature
            name: Температура

      - type: entities
        title: Гостиная
        entities:
          - entity: light.living_room_main
            name: Основной свет
          - entity: climate.living_room_ac
            name: Кондиционер
          - entity: cover.living_room_curtain
            name: Шторы

Почему иконка в боковом меню может не появиться сразу?

Home Assistant берёт иконку бокового пункта из panel_icon в config.yaml. Иногда sidebar entry кешируется. После обновления add-on стоит обновить репозиторий add-on, перезапустить add-on, удалить пункт из sidebar и добавить его снова. Если не помогает — перезапустить frontend/Home Assistant.

В настройках можно уменьшать маркеры устройств, датчики и названия помещений до 10% и увеличивать до 200%. Это полезно для маленьких экранов, горизонтальной ориентации и плотных планов.

Масштаб названий помещений применяется к подписям зон на общем плане отдельно от датчиков и маркеров.

Окно настроек стало легче: прокрутка идёт внутри одного контейнера, нижняя панель не перекрывает кнопки, а слайдеры масштаба/прозрачности не пишут состояние в /data на каждое движение пальца. Это уменьшает лаги на мобильных и в landscape-режиме.

v3.5.0: мобильные панели

На мобильных устройствах панели Комнаты и Устройства работают как компактные нижние листы. Одновременно может быть открыта только одна панель: открытие комнат закрывает устройства, и наоборот. Тап вне панели закрывает открытую панель. Прокрутка списка устройств ограничена самой панелью и не должна двигать карту или страницу.

v3.5.0: облегчённое редактирование общего плана

На мобильных и touch-экранах общий план в режиме редактирования работает в облегчённом режиме: анимации ореолов приглушаются, список устройств показывает сначала неразмещённые элементы и ограничивает количество карточек. Для поиска любого устройства используйте поле поиска. Такой режим нужен, чтобы общий план с большим количеством комнат, датчиков и устройств не лагал в landscape.

v3.5.0: устройства по комнатам в редактировании общего плана

В режиме редактирования общего плана панель Устройства теперь работает как аккордеон по комнатам. Одновременно раскрыта только одна группа, а закрытые комнаты не рендерят карточки устройств. Это снижает нагрузку на мобильных и в горизонтальной ориентации.

Есть отдельная группа Неразмещённые. Чтобы поставить устройство на общий план: откройте нужную комнату, тапните устройство, затем тапните место на карте.

v3.5.0: комнаты из Home Assistant Area

Если устройство не удалось привязать к комнате по карточке Lovelace или имени, импорт читает Home Assistant Entity/Device/Area registry через Supervisor WebSocket API и использует помещение сущности HA как fallback. Это особенно полезно для media-вкладок, где устройства лежат под heading-карточками, а не внутри обычных карточек комнат.

В режиме редактирования список устройств всегда сгруппирован по комнатам. При открытии конкретной комнаты её группа сразу раскрыта.

v3.5.0: скролл групп устройств на мобильном

В режиме редактирования панель устройств по комнатам теперь прокручивается внутри самой панели. Карточки устройств на touch больше не блокируют вертикальный свайп: для размещения используется сценарий “тап по устройству → тап по карте”.

Автоблокировка киоска

В режиме киоска можно включить Lock/Unlock. Когда киоск заблокирован, случайные тапы по устройствам, зонам и датчикам не выполняют действий. Опция Auto-lock блокирует киоск после периода бездействия, по умолчанию через 15 секунд.

v3.5.0: SVG Layout Editor без старых live-map механизмов

При входе в режим редактирования живой dashboard временно останавливается: polling состояний HA не перерисовывает экран, меню устройств, long press, glow-анимации, hover-эффекты и быстрые действия отключены. Координаты редактируются только через отдельный SVG Layout Editor: клик/тап по сетке, затем стрелки или поля X/Y, затем “Применить”. Старые live-map механизмы, прицел поверх live-карты и размещение по live-карте отключены. После “Сохранить изменения” или “Отменить изменения” live dashboard включается обратно.

v3.5.0: системные сдвоенные датчики защищены

Сдвоенные показатели температуры/влажности считаются системными. Их можно двигать и сбрасывать позицию, но нельзя удалить окончательно как обычный маркер устройства.

v3.5.0: Placement Editor Rework

Новые устройства в режиме редактирования размещаются через отдельный SVG-редактор с сеткой 0–100%, а не через живую карту. Выберите устройство, тапните точку на изображении или подстройте координаты стрелками, затем нажмите “Применить”.

v3.5.0: координаты Placement Editor

Редактор размещения теперь использует натуральные размеры изображения как SVG viewBox, а в layout сохраняет проценты. Это убирает расхождение между точкой в отдельном редакторе и положением маркера после применения, особенно на широких overview-картинках.

v3.5.0: системные датчики и debug mode

Сдвоенные датчики температуры/влажности защищены от удаления, но их можно перемещать через SVG Layout Editor: короткий тап выбирает показатель, долгое удержание открывает редактор. Координатные debug-окна скрыты по умолчанию и включаются в Настройки → Debug mode.

v3.5.0: Kiosk Attention Monitor

Функция “Внимание!” глобальная: устройство можно добавить из контекстного меню long press в любой комнате, а alert в kiosk mode показывается независимо от текущей открытой комнаты. Нормой считается состояние entity в момент включения правила. Изменение списка доступно только в admin mode и при разблокированном киоске.

Если “Зоны” включены и “Невидимые зоны” включены, зоны остаются кликабельными, но их заливка и названия скрыты. В режиме редактирования зоны всё равно показываются контурами.

v3.5.3: image converter pipeline

Загрузка общего плана стала безопаснее: сервер проверяет MIME type, расширение, реальное содержимое файла, размер файла и размер изображения. Оригинал сохраняется в /data/images/originals/, а рабочая версия оптимизируется в WebP с сохранением aspect ratio. Картинки комнат v3.5.4

В настройках можно загрузить, заменить или сбросить пользовательскую картинку каждой найденной комнаты. Комнаты не создаются вручную; список берётся из текущей конфигурации/импорта. Перед обеими операциями создаётся backup layout.json в /data/backups/.

Что делать после очистки маркеров или зон?

В v3.5.5.3 в настройках добавлены обратные действия: создание/восстановление зоны через выбор комнаты и открытие размещения устройств после очистки маркеров. Изменения применяются в SVG Layout Editor и окончательно сохраняются кнопкой “Сохранить изменения”. Функции доступны только в admin mode.

Стандартные датчики комнаты

В настройках откройте Комнаты / зоны / стандартные датчики. Для каждой найденной комнаты можно задать entity температуры, влажности, движения, шума, CO2 и освещённости.

Entity можно редактировать или очищать. Пустые поля не отображаются. Если очистить все entity комнаты, строка стандартных датчиков этой комнаты не показывается.

Полный сброс перенесён в Настройки → Layout / обслуживание, чтобы он не находился рядом с кнопкой сохранения. В нижней панели настроек добавлена кнопка Отменить настройки. Большие блоки картинок, зон, комнат/датчиков и источников открываются отдельными кнопками. FAQ исправлен для нормальной прокрутки.

Что делает “Сбросить всё к дефолту”?

Кнопка выполняет полный заводской сброс runtime-данных проекта. Перед выполнением показывается предупреждение и требуется ввести RESET. Сброс удаляет пользовательские настройки, rooms cache, устройства, layout, zones, markers, картинки, источники Lovelace/панелей, Attention, dangerous rules, command log и пользовательский PIN. Перед удалением создаётся backup. Встроенный резервный механизм восстановления владельца сохраняется.

Почему стандартные датчики на карте показываются иконками?

Чтобы строка не занимала слишком много места. На карте используются компактные иконки + значения, а в настройках остаются полные текстовые названия полей.

Раскрыта максимум одна группа; закрытые группы не рендерят карточки устройств, поэтому общий план редактируется легче. Для media-вкладок парсер учитывает heading-карточки и имя устройства: например “Алиса кабинет” попадёт в “Кабинет”, а “ТВ гостиная” — в “Гостиная”.

v3.5.0 — Выбор устройства в редакторе

В режиме редактирования список устройств больше не открывается тяжёлой панелью поверх карты. Теперь используется отдельное окно выбора устройства: нажмите “Устройства”, выберите комнату и устройство, затем тапните место на карте.

Это сделано одинаково для ПК, планшета и телефона, чтобы не было различий по платформам и ориентации экрана.

v3.5.0: Unified SVG Layout Editor

Перетаскивание полностью отключён. Новые устройства и уже размещённые маркеры редактируются одинаково: выберите устройство или маркер, откроется отдельный SVG Layout Editor, кликните/тапните новую точку на сетке, подстройте X/Y стрелками и нажмите Применить.

Live-карта больше не участвует в расчёте координат. Это убирает прилипание маркеров к курсору и ошибки, связанные с zoom/pan/hardware scale.

## v3.5.0 - Повторная очистка старого редактора: удалены пользовательские следы старого live-map редактора, старые подсказки перетаскивания и обработчики старой панели размещения. - Редактирование координат остаётся только через SVG Layout Editor. ## v3.5.0 — финальная очистка старого редактора - Удалены устаревшие пользовательские сценарии размещения поверх live-карты. - В режиме редактирования координаты меняются только через SVG Layout Editor. - Список устройств открывает редактор; маркер выбирается кликом и редактируется кликом по сетке, X/Y и стрелками. - Старое перетаскивание, live-map размещение и связанные подсказки отключены.

v3.5.0: стабильный размер SVG Layout Editor

Редактор размещения больше не использует прокручиваемый canvas и не зависит от CSS max-height внутри flex-контейнера. SVG теперь явно подгоняется JavaScript по натуральному размеру изображения и доступной области. Это должно убрать нестабильность координат в браузерах/WebView, где overflow-scroll и flex-height могли искажать координаты клика.

v3.5.0: поворот прямоугольных зон

Зоны остаются прямоугольниками. В SVG Layout Editor координаты X/Y задают центр зоны, W/H отдельно меняют ширину и высоту, а поле “Поворот” и кнопки ⟲/⟳ вращают прямоугольник вокруг центра. Подпись зоны остаётся читаемой и не поворачивается вместе с прямоугольником.

Что изменилось с профилями в v3.5.7?

Приложение начало использовать структуру профилей. Существующий проект автоматически становится первым профилем profile-1. Пользовательские картинки, layout, rooms, источники Lovelace и импортированные устройства переносятся в /data/profiles/profile-1/.

PIN, dangerous-правила, Attention Monitor и журнал команд остаются общими для всего приложения. Интерфейс управления профилями будет следующим шагом.

Как управлять профилями в v3.5.7?

Откройте Настройки → Профили. В admin mode можно создать до 3 профилей, дублировать текущий, переименовать, удалить или переключить активный профиль.

При создании нового профиля доступны опции Дублировать зоны и Дублировать значки/маркеры. Они переносят координаты зон или размещённых значков из текущего профиля в новый.

Security/PIN, dangerous rules и Attention Monitor остаются общими. Удаление профиля требует ввода DELETE, перед удалением создаётся backup. Последний профиль удалить нельзя.
